# Agent Spec - Router Agent V2

## Overview
Router Agent V2 is an enhanced version of the Router Fleet Operations Multi-Agent Orchestrator. It manages router fleet diagnostics, troubleshooting, and hardware remediation. It uses a graph-based workflow (ADK 2.0 Workflow API) to drive deterministic behavior and faster performance. All tools are served via MCP from `router-mcp-server`. The agent is fully parameterized via environment variables and uses Pydantic for data validation.

## Example Use Cases
1.  **User asks to list the fleet**: The agent enters the diagnostic node, calls `list_router_fleet` via MCP, and returns the list.
2.  **User asks to view router status**: The agent calls `render_router_card` (A2UI) via MCP and relays the JSON verbatim.
3.  **User requests a reboot**: The agent enters the remediation node, requests human confirmation (if applicable), calls `reboot_router` via MCP, and updates the UI.

## Tools Required
All tools MUST be served via the MCP server (`router-mcp-server`).
-   `list_router_fleet`
-   `get_fleet_summary`
-   `get_router_status`
-   `render_router_card`
-   `render_router_card_image`
-   `set_router_led`
-   `reboot_router`
-   `inject_bgp_fault`
-   `reset_bgp_session`
-   `send_router_command`
-   `fetch_router_logs`
-   `run_snmp_walk`
-   `clone_router`
-   `register_or_deploy_router`
-   `delete_router_node`
-   `redeploy_router_node`

> [!IMPORTANT]
> The v1 local tool `search_troubleshooting_knowledge_base` is NOT included in this list as per Requirement 2 ("All the tools must come from the MCP server"). If knowledge grounding is required, this tool must be migrated to the MCP server.

## Constraints & Safety Rules
-   Must pass all rubrics in `0-TODO/rubrics.md`.
-   High-stakes actions (`reset_bgp_session`, `reboot_router`, `inject_bgp_fault`, `send_router_command`) require explicit human confirmation.
-   A2UI JSON must be relayed verbatim without surrounding text.
-   Parameterization via `.env` file loaded into a Pydantic config class.
-   Strict use of Pydantic data classes for inputs, outputs, and state.

## Success Criteria
-   Passes `0-TODO/rubrics.md`.
-   Demonstrates faster or more deterministic routing via Graph/Workflow API compared to v1.
-   Zero manual A2UI crafting; strict relay.
-   Successful deployment to Cloud Run and registration to Gemini Enterprise.

## Architecture & Sub-Agents
-   **Root**: `Workflow` (ADK 2.0) orchestrating nodes.
-   **Nodes**:
    -   `DiagnosticNode`: Handles telemetry and status.
    -   `RemediationNode`: Handles high-stakes actions.
-   **Sub-agents**: May be used within nodes if complex reasoning is needed.
